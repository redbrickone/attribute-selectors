Hungry for "a" tags?

As presented by Chris Wright / @redbrickone

A stands for anchor

An anchor tag defines a hyperlink to a location on the same page or another page 


(def lazy-fib
 (concat
  [0 1]
  ((fn rfib [a b]
       (lazy-cons (+ a b) (rfib b (+ a b)))) 0 1)))

For Example:


          hello this is a link

Common Attributes:

  • download
  • href
  • target

Download

This is used to download a file upon clicking the link rather than navigating to another site.

<a href="https://ftp.drupal.org/files/projects/drupal-8.1.1.zip" download>Wubalubadubdub!</a>

href

Indicates the link target, either a URL or a URL fragment (generally used to show a specific part of a page)

<a href="https://developer.mozilla.org/en-US/">And awaaaay we go!</a>

hreflang

For SEO purposes - tells Google which language the page is in so it can serve your page as a result for people browsing in that language

<a href="http://www.rosettastone.com/" hreflang="en-us">Peace among worlds, Rick</a>

media

is purely advisory - to specify which kind of media query this should display on - can pass any valid media query - print, min-width, etc

<a href="https://developer.mozilla.org/en-US/">And awaaaay we go!</a>

rel

specifies the relationship of the target object and the link object

  • nofollow
  • prev
  • next
  • search

target

specifies where to display the linked resource

target

_self

Load the link in the same page - (default)

<a href="https://www.google.com/" target="_self">Just your average link</a>

target

_blank

Opens the link in a new unnamed tab

<a href="https://www.google.com/" target="_blank">Take me to a new tab!</a>

if you use this or window.open(), add rel="noopener noreferrer" or else you could be prone to viral inception in between link click and new page load

target

_parent

Loads the page in the parent browsing context of the current one. If there is no parent, it defaults back to _self

<a href="https://www.google.com/" target="_parent">Do something neat!</a>

target

_top

Load the page into the top-level browsing context (that is the browsing context that is an ancestor of the current one and has no parent) If there is no parent, it defaults back to _self

<a href="https://www.google.com/" target="_top">Do something else neat!</a>

type

Specifies the media type in the form of MIME type for the link target. Generally just advisory but could be used for media, so if someone had a speaker icon for any link with an "audio/wav" type specified

<a href="audio/song.wav" type="media/wav">Play the song (3.5 MB)</a>

ping

takes a space-separated list of URLs that are to be pinged if the user successfully navigates to the href of the hyperlink. In other words, it provides native support for click and link tracking

<a href=“#” ping=“/track/products/blaster /another/link/to/ping/to”>Click me</a>

(This is pretty redundant now that we have technologies that already do click-tracking)

referrerpolicy

A string indicating which referrer to use when fetching the resource

  • no-referrer
  • no-referrer-when-downgrade
  • origin
  • origin-when-cross-origin
  • unsafe-url
